# 베이킹 레시피 사이트

블로그·유튜브·인스타그램에서 얻은 베이킹 레시피를 모아 보는 개인용 정적 사이트.
GitHub Pages로 배포되며 빌드 과정이 없다 — 파일을 추가하고 push하면 바로 반영된다.

## 구조

- `index.html` — 사이트 전체 (목록 + 검색 + 태그 필터 + 상세 화면, 해시 라우팅)
- `recipes/*.md` — 레시피 파일 (레시피당 하나)
- `recipes/manifest.json` — 레시피 파일명 목록. **여기 등록해야 사이트에 나타난다**

## 레시피 추가 워크플로우

사용자가 레시피 링크(블로그/유튜브/인스타)를 주면:

1. **원본 읽기** — WebFetch로 링크 내용을 가져온다. 유튜브는 설명란에 레시피가 있는 경우가 많고, 없으면 자막을 시도한다. 원본을 읽을 수 없으면 사용자에게 내용을 붙여넣어 달라고 요청한다. **레시피를 지어내지 말 것.**
2. **레시피 파일 작성** — `recipes/<영문-슬러그>.md` 생성 (아래 형식 참고). 슬러그는 소문자-하이픈 영문 (예: `earl-grey-pound-cake.md`)
3. **manifest.json에 파일명 추가**
4. **커밋 & push** — 그러면 1~2분 내 사이트에 반영됨

## 레시피 파일 형식

```markdown
---
title: 레시피 이름 (한국어)
source_url: 원본 링크
source_type: youtube | blog | instagram | other
source_name: 채널명 또는 블로거 이름
tags: [파운드케이크, 오븐, 홍차]
time: 총 1시간 (+ 휴지 30분)
yield: 오란다틀 1개
date: YYYY-MM-DD (추가한 날짜)
---

## 재료

- 재료명 분량 (상태 메모: 실온, 차갑게 등)

## 만들기

1. 단계별로 번호 목록. 온도·시간 같은 핵심 수치는 **굵게**.

## 팁

- 원본에서 강조한 팁, 실패 포인트
```

작성 원칙:
- 본문은 한국어로. 원본이 외국어면 번역하되 재료 분량은 그대로 유지
- 원본에 없는 내용을 추가하지 않기. 불명확한 부분은 "(원본에 명시 없음)" 표기
- 프론트매터의 tags는 기존 레시피들과 겹치는 태그를 재사용 (태그 필터가 유용하려면 일관성이 중요)
- `recipes/basque-cheesecake.md`가 예시 파일 — 첫 실제 레시피를 추가할 때 삭제하고 manifest에서도 제거

## 확인 방법

로컬 미리보기: `python3 -m http.server 8000` 후 http://localhost:8000
(`file://`로 직접 열면 fetch가 막혀서 안 보임)
